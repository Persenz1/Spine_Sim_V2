# Spine_Sim_V2 爪刺阵列仿真实现报告

日期：2026-06-17

版本：0.2.0

本报告综合三部分依据：

1. 原始仓库实现与模型/仿真指导文档。
2. `Sipne_Claude.md` 中 Claude 的独立验收报告与修复报告。
3. 本轮终审后的追加修改、测试与复核结果。

## 1. 总体结论

当前工程已经形成一版可运行、可复现、数据优先的“推力预载下硬质粗糙表面爪刺阵列准静态仿真”实现。主物理链条为：

```text
surface bank
-> height_filtered
-> initial gaps
-> local preload W_i
-> phi_c and engagement region
-> finite-stroke first engagement search
-> finite single-spine capacity
-> displacement-controlled event loading
-> summary/spines/statistics/rankings/figures
```

硬边界仍然保持：第一版不做真实材料标定、不做对置夹持整机动力学、不让微损伤增强承载、不把自锁数学发散解释为无限承载，不把名义刺数等同于有效刺数。

本轮终审后，Claude 留下的主要未决项已经处理：

- F5：重新定义并启用有物理区分度的 `trial_force_n`。
- `cap_mode`：统一为文档枚举。
- grouped/final statistics：补齐核心分布统计和失效比例。
- schema：分析派生表按实际 DataFrame 动态写 schema。
- micro-damage：增加压力代理和可配置风险阈值接口。
- probe filter：移除均值平移，保证闭运算单调性，版本升为 v003。

测试结果：

```text

34 passed in 39.60s
```

`git diff --check` 通过。

## 2. 原始实现与 Claude 修复概览

原始 Codex 版本已经实现了主链条、surface bank、P1/P2/P3/P5/P6 管线、analysis 和 plotting，但 Claude 验收报告指出了若干工程和物理语义问题：

- P6 失败 case 写入会因非空 schema 崩溃。
- P5 没有逐 case 异常保留。
- `slip` 与 `overload` 合并，`r_slip` 恒为 0。
- 高斯探针滤波在默认分辨率下近乎失效。
- `trial_force_n=0.05` 且按名义刺数摊薄，使 `phi_c` 退化。
- grouped statistics、summary 字段、CLI 转发、Parquet row group、进度条和并行能力不完整。

Claude 修复了大部分工程问题：

- 新增 `core/parallel.py`、`core/progress.py`、`pipelines/case_tasks.py`。
- P0/P2/P3/P5/P6 改为多进程、有界在途任务和批量 Parquet 写盘。
- P5/P6 失败 case 转为失败 summary 行，不再静默丢弃或终止整轮。
- `loading.py` 按 `cap_mode` 区分 `slip`/`overload`。
- summary 增加刚度投影、LSI、压力代理等字段。
- 顶层 `spine-sim simulate/analyze/plot` 转发到真实 CLI。

本轮终审在 Claude 的基础上补齐了剩余物理/数据一致性问题。

## 3. 本轮终审修改

### 3.1 F5 临界接合角决策

文档公式为：

```text
phi_c,i = atan(F_trial_i / W_i) - phi_s
```

原实现将 `trial_force_n` 当作总试探力并除以 `n_nom`，默认 `0.05 N`，导致多数 case 中 `phi_c` 接近 `-phi_s`，实际阈值被 `phi_hook_min` 或 0 接管，`W_i -> phi_c -> engagement region` 主通路失活。

本轮修改：

- `trial_force_n` 明确定义为单刺试探切向力。
- 默认值改为 `0.50 N`，与 `F_ref_star_n` 同量级。
- 不再按名义刺数摊薄。

取舍：

- 好处：低预载刺需要更陡表面才能接合，高预载刺阈值降低，局部预载重新进入可接合区域判定。
- 代价：这是第一版的建模选择，仍不是实验标定值；正式论文结论应做 `trial_force_n` 敏感性分析。

### 3.2 cap_mode 枚举

容量模式已统一为仿真指导要求：

```text
none
no_geometric_engagement
geom_friction
strength
self_lock_strength
```

`geom_friction` 对应滑脱 `slip`，`strength/self_lock_strength` 对应强度过载 `overload`。

### 3.3 Probe Filter v003

`probe_filter_version` 升级为：

```text
morphological_closing_tip_v003
```

实现为圆盘结构元的灰度形态学闭运算。移除 Claude 版本中的均值平移，保证逐点：

```text
height_filtered >= height_raw
```

取舍：

- 好处：保留有限刺尖无法进入窄谷的物理语义。
- 代价：这是平盘闭运算近似，不是严格半球/抛物面滚球模型；但接口保持可替换。
- 影响：旧 surface bank 需要重建，下游结果不可与旧 `height_filtered` 直接混比。

### 3.4 Micro-Damage 风险接口

新增字段：

- `contact_pressure_proxy_n_per_mm2 = W_i / (pi r_t^2)`
- `damage_pressure_threshold_n_per_mm2`
- `micro_damage_risk`
- `r_micro_damage_risk`

默认不设置 `p_c`，因此不凭空给材料损伤阈值。若用户配置阈值，则只对 `W_i > 0` 的刺计算风险。

取舍：

- 好处：保留模型文档要求的 `p_i`/`D_i` 接口。
- 代价：没有材料标定时，默认风险标记为空；这比伪造阈值更稳妥。

### 3.5 Statistics 与 Schema

阶段级和最终 Monte Carlo 统计补齐核心指标分布：

```text
mean, median, std, p05, p25, p75, p95, min, max
```

覆盖字段包括承载、效率、有效刺数、载荷集中、未接触、搜索失败、行程饱和、侧接触、滑脱、过载和微损伤风险。

`stage_grouped_statistics.parquet` 和 `final_grouped_statistics.parquet` 的 schema 改为按实际 DataFrame 动态推断，避免新增统计列后 `schema.json` 滞后。

## 4. 模块实现说明

### 4.1 `surfaces/generator.py`

方法：

- 用多尺度代理形貌合成 `height_raw`。
- 按目标代理 `Rq` 归一化。
- 用 v003 形态学闭运算生成 `height_filtered`。
- 计算 raw/effective 粗糙度、坡度和几何候选密度。

取舍：

- 表面是代理模型，不是材料实测。
- 形态学闭运算强调刺尖有限半径，不做真实弹塑性接触。
- `candidate_density_preload_free` 只作 bank 审查，不替代正式接合判据。

### 4.2 `surfaces/bank.py`

方法：

- surface 生成任务可多进程并行。
- worker 直接写 NPZ，只回传统计记录。
- 主进程按 `(surface_kind, surface_id)` 排序，保证统计表顺序稳定。

取舍：

- 避免大数组跨进程传输和主进程内存堆积。
- surface bank 是长期引用源，后续 case 只保存 `surface_bank_id/surface_id`。

### 4.3 `core/types.py`

方法：

- 定义 `SingleCaseInput`、刚度模型、summary/spines/grouped/ranking/surface schema。
- 刚性阵列弹簧刚度字段保持 `null`，不写成 0。
- 新增 F5 和微损伤相关字段。

取舍：

- schema 记录“应然字段和可空性”。
- 物理 Parquet 写盘允许字段可空，以保证失败 case 可保留。

### 4.4 `io/parquet_io.py`

方法：

- `BatchedParquetWriter` 每 2000 行写一个 row group。
- 对 schema 字段进行归一化，失败行缺失字段填空。
- 物理 Arrow schema 一律放宽 nullable。

取舍：

- 批量 row group 避免十万级微 row group。
- nullable Arrow schema 优先保证长批次仿真不中断；字段约束由 `schema.json` 和测试维护。

### 4.5 `io/schema_io.py`

方法：

- 静态表使用 registry schema。
- 分析派生表使用 `dataframe_schema()` 从实际列推断 dtype、单位和可空性。

取舍：

- 静态 schema 适合核心输入/输出表。
- 动态 schema 适合统计和排名表，因为列会随分析指标扩展。

### 4.6 `solvers/contact.py`

方法：

- 先由 `height_filtered` 插值得到刺位置高度与间隙。
- 刚性阵列用数值罚刚度求预载分布，不把罚刚度作为真实弹簧参数输出。
- 柔顺阵列将轴向刚度从 N/m 转为 N/mm，并投影为 `k_nn/k_tt/k_tn`。
- 4 mm 轴向行程用于法向饱和判断。

取舍：

- 刚性求解需要数值正则化，但输出层保留刚性无真实弹簧刚度的语义。
- 柔顺模型是一阶准静态近似，未启用动态 `k_tn` 增强。

### 4.7 `solvers/engagement.py`

方法：

- `phi_c` 只在 `W_i > 0` 后计算。
- 容量按 `beta = phi_eng + phi_s` 分段：
  - `beta <= 0`：0，`no_geometric_engagement`
  - `0 < beta < 90`：`min(W_i tan beta, F_i*)`
  - `beta >= 90`：不用 `tan`，直接 `F_i*`

取舍：

- 自锁表示不滑脱，不表示无限承载。
- `F_i*` 是固定强度占位，不随当前总预载放大。

### 4.8 `solvers/search.py`

方法：

- 沿切向有限行程搜索首次接合。
- 用双线性插值取角度和高度。
- 粗步长只用于发现事件区间；穿入、接合和侧接触风险在区间内细化。

取舍：

- 第一版用 `height_filtered` 的局部坡度作为有效接触角。
- 三维几何接触角可在同一接口下替换。

### 4.9 `solvers/loading.py`

方法：

- 已接合刺在搜索距离后开始承载。
- 位移控制下直接求下一根刺达到容量的事件位移。
- 事件点总力最大值作为 `f_t_lim_n`。
- 根据 `cap_mode` 标记 `slip/overload`。

取舍：

- 不用任意小固定步长决定极限。
- 刚性阵列使用数值共享刚度，不作为材料参数解释。

### 4.10 `pipelines/p1_single_case.py`

方法：

- 单 case 主链条集中在此模块，P2/P3/P5/P6 复用它。
- 输出 `case_summary` 与 `case_spines`。
- P1 额外保存诊断数组，供绘图与人工复核。

取舍：

- P1 保存 arrays 是诊断例外；大规模阶段不保存每 case 地形数组。
- `load_success` 是默认评分成功标准，`engagement_success` 单独保留。

### 4.11 `pipelines/p2_compliant_k_alpha.py` 与 `p3_rigid_alpha.py`

方法：

- P2 扫描柔顺 `spring_k_n_per_m x alpha_p_deg`。
- P3 扫描刚性 `alpha_p_deg`。
- 每个 case 复用 P1 主求解链。
- 并行执行并批量写盘。

取舍：

- 单刺筛选用于缩小候选空间，不直接代表最终阵列表现。
- 排名引入多样性约束，避免只按均值承载力筛选。

### 4.12 `pipelines/p5_array_screen.py`

方法：

- P5a 从 P2/P3 入围单刺参数扩展到阵列行列数和间距。
- P5b 增加样本量并选出最终刚性/柔顺候选。
- 失败 case 通过 `CaseJob` 保留，不中断整阶段。

取舍：

- P5 是筛选，不是最终论文统计源。
- 保持候选多样性，避免刚性或柔顺预设胜出。

### 4.13 `pipelines/p6_final_mc.py`

方法：

- 对 P5b 最终候选执行候选、表面类别、表面样本和预载的全组合 Monte Carlo。
- 只写 `final_summary/final_spines`，不保存全二维数组。
- 生成 convergence、final ranking、sample cases 和 final report。

取舍：

- P6 是正式统计源，但仍是代理表面和未标定强度下的相对比较。
- 失败 case 保留在 summary 和 manifest 中，避免幸存者偏差。

### 4.14 `analysis/statistics.py`

方法：

- 对 candidate、surface_kind、w_total_n 分组。
- 输出完整分布统计和成功率。
- 派生 `r_uncontacted`，兼容旧数据。

取舍：

- 分组后再等权合并，避免某类表面或某档预载样本数更多时压过其他条件。

### 4.15 `analysis/ranking.py`

方法：

- P2/P3/P5 使用成功率、承载、效率、表面稳健性、搜索失败、饱和、载荷集中等指标评分。
- P2/P3 保留角度/刚度多样性。
- P5b 按角色挑选刚性 5 个和柔顺 5 个。

取舍：

- 不只按平均承载力排序。
- 排名是候选筛选工具，不是物理定律。

### 4.16 `analysis/final_mc.py`

方法：

- P6 final grouped 输出完整统计。
- final ranking 使用 P6 更大样本重新比较候选。
- P7/P8 只读取 P6 数据做表面泛化和预载效率后处理。

取舍：

- P7/P8 不重跑仿真，保证数据架构解耦。
- P6 ranking 继续保留多指标评分，不假设某一种阵列结构必然更优。

### 4.17 `plotting`

方法：

- plotting 只从已保存数据生成图片。
- P1 状态图支持 `slip/overload` 新枚举颜色。
- stage/final/P7/P8 绘图依赖分析产物。

取舍：

- 图是可再生产品，不参与仿真主流程。
- 缺字段时应由 `require_columns` 给出清晰错误。

### 4.18 CLI

方法：

- `spine-sim simulate/analyze/plot` 转发到真实 CLI。
- P0/P2/P3/P5/P6 支持 `--workers`。

取舍：

- 命令行统一，脚本入口仍可直接使用。
- 多进程使用 `forkserver/spawn`，直接调用 API 时需要 `if __name__ == "__main__"` 守卫。

## 6. 使用注意

1. 必须重建旧 surface bank，因为 `probe_filter_version` 从 v002 升到 v003。
2. `trial_force_n=0.50 N` 是当前第一版建模决策，不是实验标定值；正式论文建议做敏感性分析。
3. `damage_pressure_threshold_n_per_mm2` 默认不设置；若要输出有效 `micro_damage_risk`，需由实验或材料假设提供阈值。
4. 大规模 P6 仍建议先用小样本 smoke flow 验证完整链路，再扩大到 200000 case。
5. 当前结果适合机制比较和结构筛选，不适合作为真实墙面绝对承载预测。
