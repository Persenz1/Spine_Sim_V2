# 启动命令说明

本文说明 Spine_Sim_V2 的常用启动命令、每个启动项的含义，以及调试/正式运行时应该怎么改参数。以下命令默认在项目根目录执行：

```powershell
cd D:\Code\Spine_Sim_V2
```

PowerShell 多行命令使用反引号 `` ` `` 续行；也可以把命令写成一行执行。

## 入口命令

项目有三类脚本入口：

```powershell
python scripts/simulate.py ...
python scripts/analyze_results.py ...
python scripts/plot_results.py ...
```

- `simulate.py`：运行仿真和数据生成阶段，产生 `data/` 或 `outputs/*/data/`。
- `analyze_results.py`：读取已保存的仿真数据，重新生成统计、排名、候选清单。
- `plot_results.py`：读取数据和分析结果，生成图片；不会重新仿真。

如果已经 `pip install -e .` 安装过，也可以使用等价入口：

```powershell
spine-sim simulate ...
spine-sim analyze ...
spine-sim plot ...
```

## 启动前检查

```powershell
python scripts/simulate.py --help
python scripts/simulate.py p0-surface-bank --help
python scripts/analyze_results.py --help
python scripts/plot_results.py --help

$env:PYTEST_DISABLE_PLUGIN_AUTOLOAD="1"
python -m pytest -q
```

- `--help`：只打印参数说明，不运行仿真。
- `pytest`：跑项目测试，确认当前代码环境可用。

## 推荐小规模闭环

这组命令用于确认完整流程能跑通。每类表面只生成 3 个样本，所有阶段都用 `--workers 1`，方便排查报错。

### P0 生成 surface bank

```powershell
python scripts/simulate.py p0-surface-bank `
  --bank-id surface_bank_debug `
  --surfaces sandpaper,concrete,brick,painted_wall `
  --n-per-kind 3 `
  --resolution 5 `
  --size-x-mm 24 `
  --size-y-mm 18 `
  --tip-radius-mm 0.05 `
  --outdir data/surface_bank_debug `
  --workers 1
```

参数含义：

- `p0-surface-bank`：生成可复用的表面库。
- `--bank-id`：表面库 ID，会写入 manifest；建议和输出目录名称对应。
- `--surfaces`：要生成的表面类型。后续默认阶段要求包含 `sandpaper,concrete,brick,painted_wall` 这四类。
- `--n-per-kind`：每类表面数量。调试用 3，正式可用 1000 或更高。
- `--resolution`：每毫米网格数。越大越精细，也越慢、越占内存。
- `--size-x-mm` / `--size-y-mm`：每张表面窗口尺寸。调试可缩小，正式建议用默认/报告设置。
- `--tip-radius-mm`：刺尖半径，也决定 probe filter；改它以后需要重新生成 surface bank。
- `--outdir`：surface bank 输出目录。
- `--workers`：并行进程数。`1` 表示单进程；`0` 或省略表示自动按 CPU 核心数。

修改建议：

- 只想快速冒烟：减小 `--n-per-kind`、`--size-x-mm`、`--size-y-mm`。
- 想提高表面统计量：增大 `--n-per-kind`。
- 想重跑同一个 `--outdir`：加 `--overwrite`，或者换一个新的 `--outdir`。注意 `--overwrite` 会清理该 bank 目录内已生成内容。

### P1 单 case sanity

```powershell
python scripts/simulate.py p1-single-case `
  --surface-bank data/surface_bank_debug `
  --surface-id concrete_000000 `
  --outdir outputs/P1_single_case_sanity
```

参数含义：

- `p1-single-case`：跑一个刚性基线和一个柔顺基线，用于检查单 case 主求解链。
- `--surface-bank`：读取哪个 surface bank。
- `--surface-id`：使用哪一张表面。
- `--outdir`：P1 输出目录。
- `--compliant-spring-k-n-per-m`：柔顺基线弹簧刚度，默认 `330.0 N/m`。

修改建议：

- 想换表面：改 `--surface-id`，例如 `sandpaper_000000`。
- 想对比不同柔顺刚度：改 `--compliant-spring-k-n-per-m`。

### P2 柔顺单刺筛选

```powershell
python scripts/simulate.py p2-compliant-k-alpha `
  --surface-bank data/surface_bank_debug `
  --n-surfaces-per-kind 3 `
  --outdir outputs/P2_compliant_k_alpha_screen `
  --workers 1
```

参数含义：

- `p2-compliant-k-alpha`：扫描柔顺单刺的 `spring_k_n_per_m x alpha_p_deg x w_total_n`。
- `--surface-bank`：输入 surface bank。
- `--n-surfaces-per-kind`：每类表面取多少张参与筛选，不能超过 P0 里每类生成的数量。
- `--outdir`：P2 输出目录。
- `--workers`：并行进程数。

当前 P2 的扫描网格写在代码常量里：

- `spring_k_n_per_m = 100,150,220,330,470,680,1000`
- `alpha_p_deg = 50,60,70,80`
- `w_total_n = 0.5,1.0,1.5,2.0,2.5`

如需改这些网格，修改 `Spine_Sim_V2/pipelines/p2_compliant_k_alpha.py` 里的常量，或从 Python API 调用 `run(..., spring_values=..., alpha_values=..., w_values=...)`。

### P3 刚性单刺筛选

```powershell
python scripts/simulate.py p3-rigid-alpha `
  --surface-bank data/surface_bank_debug `
  --n-surfaces-per-kind 3 `
  --outdir outputs/P3_rigid_alpha_screen `
  --workers 1
```

参数含义：

- `p3-rigid-alpha`：扫描刚性单刺的 `alpha_p_deg x w_total_n`。
- 其他参数与 P2 类似。

当前 P3 的扫描网格主要来自代码常量：

- `alpha_p_deg = 50,60,70,80`
- `w_total_n = 0.5,1.0,1.5,2.0,2.5`

如需修改，改 `Spine_Sim_V2/pipelines/p3_rigid_alpha.py`，或从 Python API 调用 `run(..., alpha_values=..., w_values=...)`。

### P5a 阵列粗筛

```powershell
python scripts/simulate.py p5a-array-coarse `
  --surface-bank data/surface_bank_debug `
  --p2-selected outputs/P2_compliant_k_alpha_screen/data/selected_candidates.json `
  --p3-selected outputs/P3_rigid_alpha_screen/data/selected_candidates.json `
  --n-surfaces-per-kind 3 `
  --outdir outputs/P5a_array_pitch_coarse_screen `
  --workers 1
```

参数含义：

- `p5a-array-coarse`：把 P2/P3 入围的单刺候选扩展成阵列，扫描行列数和间距。
- `--p2-selected`：P2 生成的入围候选 JSON。
- `--p3-selected`：P3 生成的入围候选 JSON。
- `--n-surfaces-per-kind`：每类表面参与 P5a 的数量。

当前 P5a 的阵列网格写在 `Spine_Sim_V2/pipelines/p5_array_screen.py`：

- `rows = 2,3,4,5`
- `cols = 2,3,4,5,6,7`
- `pitch_t_mm = 4,5,6`
- `pitch_l_mm = 4,5,6`
- `w_total_n = 0.5,1.0,1.5,2.0,2.5`

如需缩小 P5a 规模，可以减少 `rows/cols/pitch` 的组合，或减少 `max_p2_selected` / `max_p3_selected`。

### P5b 阵列复筛

```powershell
python scripts/simulate.py p5b-array-refine `
  --surface-bank data/surface_bank_debug `
  --p5a-selected outputs/P5a_array_pitch_coarse_screen/data/selected_candidates.json `
  --n-surfaces-per-kind 3 `
  --outdir outputs/P5b_array_pitch_refine_screen `
  --workers 1
```

参数含义：

- `p5b-array-refine`：读取 P5a 入围候选，在更多表面上复筛。
- `--p5a-selected`：P5a 生成的入围候选 JSON。

修改建议：

- 想让 P5b 更稳：增大 `--n-surfaces-per-kind`。
- 想更快：减小 `--n-surfaces-per-kind` 或减少 P5a 入围候选数量。

### P6 最终 Monte Carlo

```powershell
python scripts/simulate.py p6-final-mc `
  --surface-bank data/surface_bank_debug `
  --selected-candidates outputs/P5b_array_pitch_refine_screen/data/selected_candidates.json `
  --n-surfaces-per-kind 3 `
  --surface-selection first_n `
  --outdir outputs/P6_final_3d_monte_carlo `
  --workers 1
```

参数含义：

- `p6-final-mc`：对 P5b 最终候选做 Monte Carlo 统计。
- `--selected-candidates`：P5b 生成的最终候选 JSON。
- `--n-surfaces-per-kind`：每类表面数量；正式运行通常是 1000。
- `--surface-selection`：表面选择策略：
  - `first_n`：每类按 `surface_id` 排序取前 N 个，完全确定。
  - `random_fixed`：用固定随机种子抽 N 个。
  - `explicit_list`：使用用户给定的表面列表。
- `--surface-list`：当 `--surface-selection explicit_list` 时使用，可传 JSON 文件或逗号分隔的 `surface_id`。
- `--random-seed`：当 `random_fixed` 时使用，默认 `20260617`。

修改建议：

- 调试用 `first_n + 小 n`。
- 正式随机抽样用 `random_fixed`，并记录 `--random-seed`。
- 想复现实验子集，用 `explicit_list` 固定表面 ID。

## 运行规模怎么估算

P0 表面数：

```text
表面总数 = 表面类型数 x n-per-kind
```

P2 case 数：

```text
4 类表面 x n-surfaces-per-kind x 7 个弹簧刚度 x 4 个 alpha x 5 个预载
```

P3 case 数：

```text
4 类表面 x n-surfaces-per-kind x 4 个 alpha x 5 个预载
```

P5/P6 case 数：

```text
候选数量 x 4 类表面 x n-surfaces-per-kind x 5 个预载
```

每个 case 还会输出一行 summary 和多行 spines。阵列越大，`stage_spines.parquet` / `final_spines.parquet` 越大。

## 常见修改方式

### 只验证程序能启动

用 `--help` 和 `pytest`，不生成数据。

### 快速跑通完整流程

保持：

```text
P0 --n-per-kind 3
P2/P3/P5/P6 --n-surfaces-per-kind 3
--workers 1
```

### 中等规模调参

建议：

```text
P0 --n-per-kind 30 到 100
P2/P3 --n-surfaces-per-kind 10 到 30
P5a --n-surfaces-per-kind 10 到 30
P5b --n-surfaces-per-kind 30 到 100
P6 --n-surfaces-per-kind 100 到 300
--workers 0
```

### 正式统计

建议：

```text
P0 --n-per-kind 1000 或更高
P6 --n-surfaces-per-kind 1000
--workers 0
```

正式结果要记录：

- 代码版本。
- `surface_bank_id`。
- `--random-seed`。
- 每个阶段的 `manifest.json`。
- 每个阶段的 `selected_candidates.json`。

## 后处理命令

仿真完成后，可以单独重新分析和绘图。
P2/P3/P5/P6 的 `simulate` 命令在写完 case 数据后仍会自动运行分析后处理；该阶段会显示独立进度，并只读取统计/排名需要的 summary 列以降低内存占用。

```powershell
python scripts/analyze_results.py stage `
  --stage-dir outputs/P2_compliant_k_alpha_screen

python scripts/analyze_results.py final `
  --stage-dir outputs/P6_final_3d_monte_carlo

python scripts/analyze_results.py p7-surface `
  --p6-dir outputs/P6_final_3d_monte_carlo `
  --outdir outputs/P7_surface_generalization

python scripts/analyze_results.py p8-preload `
  --p6-dir outputs/P6_final_3d_monte_carlo `
  --outdir outputs/P8_preload_efficiency
```

绘图：

```powershell
python scripts/plot_results.py surface-audit `
  --surface-bank data/surface_bank_debug `
  --sample-per-kind 2 `
  --outdir outputs/P0_surface_audit `
  --style report

python scripts/plot_results.py p1 `
  --stage-dir outputs/P1_single_case_sanity `
  --style debug

python scripts/plot_results.py stage `
  --stage-dir outputs/P2_compliant_k_alpha_screen `
  --style report

python scripts/plot_results.py final `
  --stage-dir outputs/P6_final_3d_monte_carlo `
  --style report
```

`--style` 可选：

- `debug`：调试图。
- `report`：报告图。
- `paper`：论文/正式风格图。

## 容易踩坑的地方

- P2/P3/P5/P6 的 `--n-surfaces-per-kind` 不能超过 P0 生成的 `--n-per-kind`。
- 改了 `--resolution`、`--size-x-mm`、`--size-y-mm`、`--tip-radius-mm` 后，需要重新生成 surface bank。
- 后续阶段默认依赖四类表面：`sandpaper,concrete,brick,painted_wall`。如果 P0 少生成某一类，后续筛选会报错。
- P5a 依赖 P2/P3 的 `selected_candidates.json`；P5b 依赖 P5a 的 `selected_candidates.json`；P6 依赖 P5b 的 `selected_candidates.json`。
- `--workers 0` 跑得快，但并行报错时不如 `--workers 1` 好定位。
- `data/` 和 `outputs/` 通常会很大，正式运行前确认磁盘空间。
